# Trabajo Práctico — Programación 1

## Objetivo

El objetivo de este trabajo práctico es aplicar conceptos básicos de programación en Python, haciendo foco en:

- Estructuras de control de flujo
  - Condicionales (`if`, `elif`, `else`)
  - Bucles (`for`, `while`)
- Tipos de datos compuestos (Listas, tuplas, diccionarios, sets)
- Funciones y modularización del código
- Resolución de problemas mediante algoritmos simples
- Buenas prácticas de desarrollo utilizando Git y GitHub

---

## Estructura del repositorio

Cada ejercicio se encuentra en un archivo independiente con el siguiente formato:

```text
ejercicio_XX.py
```

Por ejemplo:

```text
ejercicio_01.py
ejercicio_02.py
ejercicio_03.py
```

Inicialmente, cada archivo contiene únicamente la consigna del ejercicio dentro de un comentario.

Cada alumno deberá completar el código correspondiente en cada archivo.

---

## Modalidad de trabajo

Cada alumno deberá trabajar utilizando Git y GitHub siguiendo el flujo indicado a continuación.

### 1. Crear una rama personal

Cada alumno debe crear su propia rama a partir de `main`.

La rama deberá seguir la siguiente nomenclatura:

```text
apellido_nombre
```

---

### 2. Resolver los ejercicios

Cada ejercicio debe resolverse directamente en su archivo correspondiente.

---

### 3. Realizar commits

Se debe realizar:

- Al menos **un commit por ejercicio**
- Commits adicionales si es necesario corregir o mejorar una solución

Se recomienda escribir mensajes de commit claros y descriptivos.

Ejemplos:

```text
"Resuelve ejercicio 1"
"Corrige ejercicio 3"
"Agrega validación en ejercicio 5"
```

---

### 4. Crear un Pull Request (PR)

Cada alumno deberá crear un Pull Request (PR) hacia la rama `main`.

El PR será utilizado para:

- Revisar el código entregado
- Realizar observaciones y correcciones
- Sugerir mejoras
- Evaluar la utilización de Git y GitHub

Los comentarios y correcciones deberán resolverse directamente sobre la misma rama del alumno.

---

## Recomendaciones

- Mantener el código ordenado y legible
- Utilizar nombres de variables descriptivos
- Probar cada ejercicio antes de realizar el commit
- Realizar commits pequeños y frecuentes

---


## Entrega

La entrega se considera realizada una vez que:

- Todos los ejercicios solicitados estén resueltos
- Los cambios estén subidos al repositorio remoto
- El Pull Request haya sido creado correctamente
